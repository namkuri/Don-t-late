# 아트 모드 (민지) — 생성·반입 셀프서비스

> 확정: [decisions](../decisions.md) D-055 (2026-07-22). 민지는 Unity 없이도 돌아가는 게 원칙 —
> 반입하면 검역·배치·프리뷰는 파이프라인(관제)이 처리한다. 산출물 반입도 **브랜치→PR** (main 차단).

## 0. 최초 셋업 (1회)

1. clone 후 `git config core.hooksPath hooks` (라이선스 대조·main 차단 훅 활성)
2. `dontlate.role`은 설정하지 않는다 — main push 자동 차단(브랜치→PR만).
3. Unity는 필수 아님 — 결과 확인은 프리뷰 회신(아래 §3)으로.

## 1. 뭘 만들지 — 작업 카탈로그

- 정본: [BOM.md](../BOM.md) (규격·치수·폴리 상한) + [INBOX](../INBOX.md)의 H-항목(재요청).
- 현재 열린 것 예: H12 편의점 store_2 데시메이트(<3,000tris)+텍스처 · 가로등(<1,500tris)+텍스처 · H9 idle 애니.
- 공통 규격: **FBX Embed Media 필수** · 원점=바닥중심 · 실상표 금지 · min-BOM은 파일명=bom_id ·
  간판 있는 건물은 **간판 분리 익스포트**(별도 메시/머티리얼 — 밤 발광용).

## 2. 반입 절차

```
브랜치 생성 → Assets/_intake/art/<도구명>/<분류>/ 에 파일 추가 → 커밋·push → PR
```
- 폴더 예: `_intake/art/Trellis2/Buildings/` · `_intake/art/Hand/` (수제)
- PR 본문에: 도구·플랜(라이선스 판단 근거)·원파일명·의도한 bom_id 한 줄씩.
- 라이선스·출처 누락은 실격 사유 영역 — 반입 차단(타협 없음).

## 3. 회신 (관제가 PR에 남기는 것)

- **검역 리포트**: 폴리 실측·크기·텍스처 유무·경고/차단 (경고는 반입 진행, 개선 재요청만).
- **프리뷰 스크린샷**: 씬에 꽂힌 낮/밤 컷 — Unity 없이 인게임 룩 확인.
- 판정 어휘: ACCEPT / 조건부 ACCEPT(치유 조건) / REJECT(사유) — merge는 관제(남규).

## 4. 팁 (실사고에서 나온 규칙)

- 텍스처가 안 보이면 십중팔구 Embed Media 꺼짐 (가로등·캐릭터 실사례 — 회색 렌더).
- 축이 눕는 문제는 익스포트 시 Y-up 확인.
- 수제 저폴리는 파이프라인 최고 우량 반입물 (box.fbx 106tris — 유일한 예산 통과 사례).

## 5. 디스코드 알림 (선택)

- #git 채널에 발주(📦)·검역 결과·PR 알림이 흐른다 — 반입 후 회신은 PR 코멘트+디스코드로 온다.
