## 1. 명세할 것 (관제 몫 — 문서로 못박기)

### 1-1. 구역별 소요 리스트 (BOM 확장) — **구역 컨셉 확정(§2-1) 후에**
- 형식: 구역 × {근경(카메라 앞 소품) / 중경(도로·보도·건물 전면) / 원경(스카이라인 실루엣)} 3레인 표.
- 각 셀에 bom_id·수량·폴리 타겟·낮밤 베리에이션 여부. → [BOM](BOM.md) §신설로 편입, 발주는 이 표에서 pull.

### 1-2. 임포트 계약 재확인 (기존 그대로 — 파이프라인 출구 규격)
- 경로: `Assets/_intake/art/<도구>/{Buildings|Props|Characters|...}` → 검역 → `Art/…` (자동 임포트 트리거).
- 파일명 = bom_id · 원점 = 바닥중심 · **fbx는 유니티에서 scale (1,1,1)이어야 함** (민지 채팅 그대로 — 블렌더 익스포트 시 Apply Transform).
- 텍스처 **Embed Media 필수** (전과 사고 2회).

### 1-3. 스케일·폴리 규격표 (충돌 정리 — 이것이 정본)

| 분류 | 높이/치수 | 폴리 타겟 (리메시 목표) |
|---|---|---|
| 캐릭터 | **1.8u** (채팅 1.7u는 폐기 — 기존 앵커·빌더 정규화가 1.8) | <5,000 |
| 건물 | **층수 × 3.0u** (1층 상가 3u · 3층 빌라 9u — 채팅 "4u"는 1.5층엔 맞고 일반화 불가) | <3,000 |
| 소품(담장·표지판·전봇대 등) | 실물 비례 (1u=1m) | <1,500 |
| 바닥 타일 텍스처 | 1타일 = 1u 권장 | 256px 이하 (임포터가 강제) |

## 2. 기획할 것 (사람 결정 — 남규 or 남규+민지 상의)

1. **구역 컨셉 2+α 확정**: 현행 구역은 "행복빌라 구역"(근거리)·"달빛맨션 구역"(원거리) 2개.
   채팅의 시가지/도심/먹자골목을 **기존 2구역의 컨셉으로 입히든가**(추천 — 시스템 무변경),
   3번째 구역 추가든가(TravelMapView 노드+주문 풀 확장 필요 — 관제 1발주 거리). **← 이 결정이 §1-1 명세의 선행 조건.**
2. **구역 선택 화면**: 현행 Travel = 2D 노드 버튼. 2D 일러스트 맵 + UI(추천 — 아트 1장이면 됨, 컨셉아트 겸용) vs 3D 미니어처(비쌈).
3. **컨셉 아트**: 구역별 1장 (ChatGPT 이미지 라인 — UI 6종과 같은 구두 계약으로 가능). 결정되면 A-004로 발주.

## 3. 전략 짤 것 (민지 집중 — 모델 생성 파이프라인)

> 목표 체인: 생성(Trellis/Tripo) → **블렌더 헤드리스 자동 변환**(리메시→폴리 타겟·텍스처 다운샘플→스케일 정규화→fbx) → _intake 반입 → 유니티 임포터(검증·프리팹화 — 기존).

- 단계 제안 (작게 관통 먼저):
  1. **로컬 블렌더 CLI로 변환 스크립트 1개 관통** — 소품 1개: 리메시+다운샘플+Apply Transform+fbx 익스포트. RunPod 없이 파이썬 `bpy` 스크립트부터. ← **H4와 합류, 최우선**
  2. 스크립트가 돌면 → RunPod 리눅스에 Blender headless 설치, 같은 스크립트 실행 (환경만 이식).
  3. 그 다음에야 ComfyUI 커스텀 노드로 감싸기 (생성→변환 원버튼).
  - 순서 근거: ComfyUI 노드부터 만들면 디버깅이 3층(노드↔RunPod↔블렌더)을 관통해야 한다 — 변환 스크립트가 정본이고 나머지는 포장.
- 관제 지원: 출구 검증은 기존 임포터가 이미 수행(스케일·폴리 경고·프리팹 자동) — 파이프라인은 **§1-3 타겟값만 맞추면 됨**. 변환 스크립트 리포 반입 시 `scripts/blender/` 예약.
