# 게임 BGM 공용 규격

> 스타일 md(`afternoon-bgm-*.md`, `night-bgm.md`)와 **함께** 읽는다.
> 여기서 정한 것이 스타일 기준과 충돌하면 **이 문서가 이긴다.**
> 스타일 md는 "어떤 소리인가", 이 문서는 "게임에 들어갈 수 있는가"를 다룬다.

---

## 0. 전제 — 프롬프트가 할 수 있는 일의 한계

산출물은 감상용 곡이 아니라 **게임 내 상시 반복 BGM**이다.
프롬프트의 역할은 완성품을 뽑는 게 아니라 **편집 비용을 낮추고 반복 내성을 확보하는 것**이다.

| 항목 | 프롬프트로? | 실제 해결처 |
|---|---|---|
| BPM 지정 | 🟢 가능 | — |
| 페이드인·아웃 억제 | 🟢 가능 | 프롬프트 + 편집 |
| 반복 내성 | 🟢 프롬프트가 주역 | — |
| 다이내믹 평탄화 | 🟢 가능 | 프롬프트 + 컴프 |
| 중음역 비우기 | 🟡 부분적 | 프롬프트 + EQ |
| 마디 경계 정렬 | 🟡 BPM까지만 | **편집** |
| 심리스 루프 | ❌ 불가 | **편집 전용** |
| 러드니스 통일 | ❌ 불가 | **편집 전용** |
| 포맷·인코더 패딩 | ❌ 불가 | **내보내기 설정** |

❌ 항목은 아무리 프롬프트를 다듬어도 안 된다. 파형 편집기에서 처리한다.

---

## 1. 필수 포함 태그

프롬프트 뒤쪽(스타일 태그 다음, BPM 앞)에 배치한다.

```
no intro fade-in, no outro fade-out, no ending,
continuous groove from start to finish,
consistent dynamics, steady level, no dramatic build-ups, no drops,
background music for a game, understated melody, texture-driven, unobtrusive,
minimal arrangement, sparse midrange, leave space in the mids
```

### 각 줄의 이유

| 줄 | 막는 문제 |
|---|---|
| `no intro/outro fade`, `no ending` | AI가 붙이는 페이드·종결 코드 → 루프 3대 적 |
| `continuous groove` | 곡 중간이 비어 잘라 쓸 구간이 줄어드는 것 |
| `consistent dynamics`, `no drops` | 음량 기복 → 효과음이 묻히고 이음매가 튐 |
| `background music`, `understated` | 30회 반복 시 훅이 고문이 되는 것 |
| `sparse midrange` | 대사·효과음이 몰린 1~4kHz 충돌 |
| `minimal arrangement` | 레이어 많을수록 이음매 어긋남·피로 증가 |

---

## 2. 필수 명시 항목

| 항목 | 규칙 | 이유 |
|---|---|---|
| **BPM** | **정수 1개.** 범위 금지 (`100-110` ❌ / `104` ⭕) | 마디 계산의 유일한 근거. 범위면 실측을 다시 해야 한다 |
| **조성** | `major key` 또는 `minor key` 명시 | 무드의 가장 강한 손잡이 |
| **길이** | **목표 루프의 2배 이상** (기본 3분) | 편집은 덜어내는 작업. 재료가 딱 맞으면 실패한다 |
| **instrumental** | 항상 | — |

### 목표 루프 길이 기준

| 루프 | 30분 플레이 시 반복 | 판정 |
|---|---|---|
| 60초 | 30회 | ❌ 짧다 |
| 90초 | 20회 | 🟡 최소선 |
| 2~3분 | 10~15회 | ⭕ 권장 |

---

## 3. 금지 태그 (게임 BGM 한정)

```
catchy hook, memorable melody, anthemic, energetic build,
epic ending, climax, final chord, big finish
```

**감상용 기준으로는 미덕이지만 상시 BGM에서는 부채다.**

> ⚠ 예외: 타이틀 화면·트레일러·엔딩용은 훅이 **있어야** 한다.
> 그런 용도는 이 문서를 적용하지 말고 스타일 md만 쓴다.
> 프롬프트를 짤 때 **용도를 먼저 확정**한다 — 상시 BGM인가, 단발 연출인가.

---

## 4. 음색 선택의 대가 — 루프 난이도

무드를 위해 쓰는 태그가 편집 난이도를 결정한다. **금지가 아니라 비용 고지**다.

| 태그 | 루프 난이도 | 이유 |
|---|---|---|
| `gated reverb` | 🟢 쉬움 | 잔향을 칼로 끊어 꼬리가 없다 |
| `dry`, `tight`, `punchy` | 🟢 쉬움 | 잔향 짧음 |
| `plate reverb (short)` | 🟡 보통 | — |
| `hazy`, `dreamy`, `spacious` | 🔴 어려움 | 긴 잔향 꼬리가 자르는 지점에서 싹둑 끊긴다 |
| `cavernous reverb`, `long tail` | 🔴 어려움 | 위와 동일, 더 심함 |
| `vinyl warmth`, `tape hiss`, `dusty`, `lo-fi` | 🔴 어려움 | 지속 노이즈가 이음매에서 불연속을 만든다 |

🔴 태그를 쓰면 프롬프트 하단에 다음을 남긴다:

```
⚠ 편집 경고: 긴 잔향/지속 노이즈 포함. 루프 이음매에 테일 랩 필요.
```

**우회로**: 밤·몽롱 무드가 필요한데 루프가 안 잡히면, 밤 전용 곡을 새로 뽑지 말고
**낮 곡에 로우패스 + 리버브를 걸어 변주**한다. 곡 1개로 두 시간대를 처리할 수 있고
정체성도 유지된다.

## 5. 출력 포맷 — 프롬프트는 편집 지시서를 겸한다

프롬프트를 출력할 때 **아래 블록을 반드시 함께** 낸다.
곡을 받는 순간 "어디를 자를지"가 이미 손에 있어야 한다.

```
── 편집 인계 ──
BPM        : 104
1마디      : 2.31초   (240 ÷ BPM)
권장 루프  : 32마디 = 73.85초   (대안: 16마디 = 36.92초)
요청 길이  : 180초
편집 경고  : (있으면 §4 문구)
```

### 마디 길이 표 (자주 쓰는 BPM)

| BPM | 1마디 | 16마디 | 32마디 |
|---:|---:|---:|---:|
| 84 | 2.86s | 45.71s | 91.43s |
| 88 | 2.73s | 43.64s | 87.27s |
| 92 | 2.61s | 41.74s | 83.48s |
| **96** | **2.50s** | **40.00s** | **80.00s** |
| 100 | 2.40s | 38.40s | 76.80s |
| 104 | 2.31s | 36.92s | 73.85s |
| 108 | 2.22s | 35.56s | 71.11s |
| 112 | 2.14s | 34.29s | 68.57s |
| 116 | 2.07s | 33.10s | 66.21s |
| **120** | **2.00s** | **32.00s** | **64.00s** |

> **96·120 BPM은 초 단위가 딱 떨어진다.** 편집이 가장 쉽다.
> 무드가 허락하면 이쪽으로 붙여라.

## 6. 산출물 검증 체크리스트

프롬프트를 고쳤으면 **결과를 재야** 한다. 안 재면 경험으로 같은 걸 계속 재발명한다.

| # | 측정 | 방법 | 통과선 |
|---|---|---|---|
| 1 | BPM 준수 | 탭 템포 + 파형, 두 방법 일치 | 지정값과 동일 |
| 2 | 페이드 유무 | 파형 앞뒤 10초 육안 | 페이드 없음 |
| 3 | 유효 구간 | 쓸 수 있는 연속 길이 | 목표 루프 × 1.5 이상 |
| 4 | 잔향 꼬리 | 마디 경계에서 잘라 이음매 청취 | 싹둑 소리 없음 |
| 5 | 반복 내성 | 5분 무한 반복 청취 | 3분까지 안 거슬림 |

**3개 이상 통과 → 그 프롬프트는 게임용으로 쓸 만하다.**
2개 이하 → 프롬프트를 고친다. 곡을 다시 뽑는 게 아니라 **프롬프트를 고친다.**

## 7. 편집 단계 (프롬프트 밖 — 참고용)

프롬프트가 아무리 좋아도 이 단계는 건너뛸 수 없다.

```
1. BPM 실측            → 1마디 초 = 240 ÷ BPM
2. 첫 강박 위치 찾기    → 0:00이 아니다
3. 루프 길이 결정       → 16 또는 32마디
4. 마디 경계에서 자르기
5. 이어붙여 이음매 청취
6. 이음매 수리
   · 딱 소리(클릭)  → zero crossing 맞추기 / 1~5ms 초단 페이드
   · 잔향 끊김      → 테일 랩 (루프 끝 이후 잔향을 시작 부분에 겹쳐 믹스)
   · 화성 안 맞물림 → 마디 계산 오류. 3번으로
7. 러드니스 정규화      → 전 곡 동일값
8. 내보내기            → WAV 또는 Ogg Vorbis
```

> ⚠ **mp3 금지.** 인코더가 앞뒤에 무음 패딩을 강제로 붙여 매 루프마다 공백이 생긴다.
> ⚠ **크로스페이드 주의.** 끝과 시작을 겹치면 루프가 그만큼 짧아져 박자가 밀린다. 테일 랩을 써라.

## 부록 — 최소 용어

| 용어 | 뜻 |
|---|---|
| **BPM** | 분당 박 수. `1박 = 60÷BPM초` |
| **마디(bar)** | 4박 묶음(4/4). `1마디 = 240÷BPM초` |
| **프레이즈** | 음악의 자연 단락. 4·8·16·32마디 |
| **첫 강박(downbeat)** | 곡의 첫 1박. 파일 0.0초가 아니다 |
| **zero crossing** | 파형이 0을 지나는 지점. 여기서 자르면 클릭이 안 난다 |
| **테일 랩** | 루프 끝 이후의 잔향을 루프 시작에 겹쳐 믹스하는 기법 |
| **LUFS** | 사람이 느끼는 음량 단위. 피크와 다르다. 곡 간 음량 통일의 유일한 객관 척도 |
| **트루피크** | 실제 최대 진폭. -1dB 여유를 남긴다 |
| **스템** | 악기별로 분리된 트랙. 수직 레이어링에 필요 |
