# 게임 SFX 공용 규격

> `GAME-BGM-RULES.md`의 자매 문서 — **SFX(단발 효과음)에 한해 이 문서가 제1지침이다.**
> BGM 규격은 SFX에 적용하지 않는다 (BPM·조성·루프 마디 계산은 SFX에 무의미).
> 개별 프롬프트(`prompts/sfx_*.md`)의 창작 태그와 충돌하면 **이 문서가 이긴다.**
> 목록·트리거·길이의 정본은 `planning/BOM.md` §8 — 이 문서는 "어떻게 뽑는가"만 다룬다.

---

## 0. 전제 — SFX는 BGM과 반대 방향의 물건이다

BGM은 **30회 반복돼도 안 거슬리는 배경**이 목표다.
SFX는 **바쁜 믹스 위에서 0.1초 안에 읽히는 신호**가 목표다.

| | BGM | SFX |
|---|---|---|
| 목표 | 존재감 지우기 | 존재감 세우기 |
| 주파수 자리 | 중음역 비우기 (`sparse midrange`) | **비워진 1~4kHz를 차지** |
| 반복 | 같은 파일 무한 루프 | 같은 파일 **연타** (픽업·발소리·판정음) |
| 최대 적 | 훅·페이드·다이내믹 기복 | **앞 무음·긴 꼬리·배경음 혼입** |

> BGM 규격 §1의 `sparse midrange`는 SFX를 위한 자리 계약이다.
> SFX가 그 자리(1~4kHz 중심)를 벗어나면 계약이 무효가 된다.

### 프롬프트가 할 수 있는 일의 한계

| 항목 | 프롬프트로? | 실제 해결처 |
|---|---|---|
| 소리 정체(재질·동작) | 🟢 가능 | — |
| 단발성 (음악·보컬 배제) | 🟢 가능 | — |
| 잔향 억제 | 🟢 가능 | 프롬프트 + 편집 |
| 대략 길이 | 🟡 "about N seconds"까지 | **트림** |
| 시작 무음 제거 | ❌ 불가 | **트림 전용** |
| 정확한 길이 | ❌ 불가 | **트림 전용** |
| 러드니스 통일 | ❌ 불가 | **정규화 전용** |

❌ 항목은 프롬프트를 아무리 다듬어도 안 된다. 파형 편집기에서 처리한다.

## 1. 필수 포함 태그

프롬프트 뒤쪽(창작 태그 다음)에 배치한다. 현행 조립기(`prompt_builder.py`)가 붙이는 문구와 동일 —
손으로 프롬프트를 짤 때도 이 블록은 그대로 쓴다.

```
Single isolated sound effect, dry and close,
no background music, no vocals, no long reverb tail
```

추가로 스타일 앵커:

```
retro pixel-art game sound design, dark comedy tone,
clean and readable in a busy mix
```

### 각 줄의 이유

| 줄 | 막는 문제 |
|---|---|
| `Single isolated sound effect` | AI가 앰비언스·배경 레이어를 깔아 오는 것 |
| `dry and close` | 공간 잔향 → 연타 시 잔향이 겹쳐 머드가 되는 것 |
| `no background music`, `no vocals` | 음악·목소리 혼입 = 즉시 실격 (금칙 검사 대상) |
| `no long reverb tail` | 꼬리가 다음 재생과 겹침 · 트림 지점이 사라짐 |
| `clean and readable in a busy mix` | BGM 위에서 안 읽히는 소리 |
| `dark comedy tone` | 톤 근거 = INTENT(동결): 다크코미디 · "늦지마!!" — 과장·만화적 ⭕, 실사 잔혹 ❌ |

---

## 2. 필수 명시 항목

| 항목 | 규칙 | 이유 |
|---|---|---|
| **길이** | `Duration about N seconds` — N은 BOM §8 표 값 (기본 0.1~1.5s) | 과길이 = 크레딧 낭비 + 트림 노동 |
| **단발/루프** | 기본 단발. **루프는 `amb_night` 단 1종** — 루프면 `seamless loop, ambient bed` 명시 | 루프 여부가 편집 공정을 가른다 |
| **동작 서술** | 소리의 **원인 동작**을 쓴다 (`box being grabbed` ⭕ / `pickup sound` ❌) | 추상어는 UI 삑삑이가 나온다 |
| **레이어 수** | 1~2겹 서술까지 (`scuff + soft thud`) | JUICE "작은 순간 1~2레이어" — 3겹부터 흐려진다 |

### 쌍·계열 규칙

`sfx_rhythm_hit / _miss` 같은 **판정 쌍은 같은 음색 계열로 뽑고 피치·방향만 대비**시킨다
(히트=상행·밝음 / 미스=하행·둔탁). 계열이 다르면 순간 판별이 안 된다.
연타 전제 SFX(픽업·발소리·판정음)는 특히 §1의 `dry` 준수 — 꼬리가 겹치면 연타가 뭉갠다.

## 3. 금지 태그 (게임 SFX 한정)

```
cinematic, epic, trailer, riser, braam,
music, melody, jingle, chord progression,
crowd, voice, speech, scream,
huge reverb, cavernous, distant
```

**영화 예고편 기준으로는 미덕이지만 0.5초 게임 신호에서는 부채다.**

> ⚠ 예외: `sfx_scene_whoosh`(전환음)는 `riser` 계열 허용 — 단 1.5s 상한 유지.
> `amb_night`는 애초에 신호가 아니라 배경이므로 `distant` 허용.
> 프롬프트를 짤 때 **신호인가, 배경인가**를 먼저 확정한다 (BGM 규격 §3의 용도 확정과 같은 원리).

---

## 4. 음색 선택의 대가 — 연타 난이도

금지가 아니라 **비용 고지**다. BGM의 루프 난이도에 대응하는 SFX의 축은 **연타 겹침**이다.

| 태그 | 연타 난이도 | 이유 |
|---|---|---|
| `dry`, `tight`, `short`, `percussive` | 🟢 쉬움 | 꼬리 없음 — 겹쳐도 깨끗 |
| `metallic ring`, `bell-like` | 🟡 보통 | 자연 감쇠 꼬리 — 트림 지점 판단 필요 |
| `reverb`, `echo`, `washy` | 🔴 어려움 | 연타 시 꼬리 중첩 → 머드 · 트림 불가 지점 |
| `rumble`, `sub bass` | 🔴 어려움 | 저역 중첩 → WebGL 소출력에서 왜곡 · 1~4kHz 계약 이탈 |

🔴 태그를 쓰면 프롬프트 하단에 남긴다:

```
⚠ 편집 경고: 잔향/저역 포함. 연타 트리거면 꼬리 트림 + 하이패스 필요.
```

## 5. 출력 포맷 — 프롬프트는 반입 지시서를 겸한다

프롬프트를 출력할 때 **아래 블록을 반드시 함께** 낸다.
파일을 받는 순간 "어디를 자르고 어디에 놓을지"가 이미 손에 있어야 한다.

```
── 반입 인계 ──
bom_id     : sfx_pickup            (파일명 = bom_id — 스왑 계약)
트리거     : PackagePickedUp       (BOM §8 표)
요청 길이  : 1.5s (트림 후 목표: 앞 무음 0 · 꼬리 자연 감쇠까지)
착지       : Assets/_intake/ElevenLabs/SFX/sfx_pickup.wav
최종 dest  : Assets/Audio/SFX/sfx_pickup.wav  (같은 파일명이면 합성 폴백 자동 교체)
편집 경고  : (있으면 §4 문구)
```

### 기술 규격 (BOM §8 · D-040 실측 — 프롬프트 밖 고정값)

| 항목 | 값 | 근거 |
|---|---|---|
| 생성 출력 | `pcm_44100` → WAV 래핑 | mp3 금지 — 인코더 무음 패딩 (BGM 규격 §7과 동일) |
| Unity 임포트 | Vorbis **q70** · **Decompress On Load** · **모노(2D)** | 임포터 자동 적용 — 수동 설정 금지 |
| 총 예산 | BGM 포함 ≤ 10MB (미검증 추정 — D-044) | 컷은 예산이 아니라 품질로 판정 |
| 라이선스 | 생성 즉시 `Assets/Audio/CREDITS.md` append | **누락 = 반입 차단, 실격 사유 영역** |

## 6. 산출물 검증 체크리스트

프롬프트를 고쳤으면 **결과를 재야** 한다.

| # | 측정 | 방법 | 통과선 |
|---|---|---|---|
| 1 | 앞 무음 | 파형 앞부분 육안 | 트림 후 **0ms** (트리거 즉시성 — 버튼 느낌) |
| 2 | 혼입 | 청취 | 음악·보컬·배경 앰비언스 없음 (있으면 즉시 재생성) |
| 3 | 길이 | 트림 후 실측 | BOM 표 값 이내 |
| 4 | 연타 내성 | **5연타 재생** 청취 (연타 트리거만) | 꼬리 겹침 머드 없음 |
| 5 | 믹스 가독 | BGM(0.5) 위에 SFX(0.7)로 재생 | 한 번에 읽힘 — 안 들리면 볼륨이 아니라 **주파수 자리**를 의심 |

**4개 이상 통과 → 반입.** 2·5번 실패 → 프롬프트를 고친다 (창작 태그의 재질·자리부터).
1·3번 실패 → 프롬프트 잘못이 아니다. **트림한다.**

> 유료 구간 1회 제한(파이프라인 정책)과 정합: 자동교정 루프는 무료 구간(프롬프트 검사)까지만 —
> 생성 후 실패는 사람 판정을 거쳐 재생성한다.

## 7. 편집 단계 (프롬프트 밖 — 참고용)

BGM의 루프 공정보다 훨씬 짧다. 그래서 SFX는 물량이 가능하다.

```
1. 앞 무음 트림        → 첫 트랜지언트 직전까지 (zero crossing에서)
2. 꼬리 정리           → 자연 감쇠 끝 또는 BOM 길이 상한에서 짧은 페이드아웃(5~20ms)
3. 러드니스 확인       → SFX끼리 상대 균형 (판정음 쌍은 동일 레벨)
4. 트루피크            → -1dB 여유
5. 내보내기            → WAV (임포터가 Vorbis 변환)
```

## 부록 — 최소 용어 (BGM 규격 부록에 없는 것만)

| 용어 | 뜻 |
|---|---|
| **트랜지언트** | 소리의 타격 순간(어택). SFX 인지성의 몸통 — 앞 무음 트림의 기준점 |
| **머드** | 중저역이 겹쳐 흐려진 상태. 연타 꼬리 중첩의 결과 |
| **하이패스** | 저역 걷어내는 필터. 저역 중첩·1~4kHz 계약 이탈의 수리 도구 |
