# PRETASK.md — 사전과제 모드 헌장
## 같은 레일, 반대 모드: 제출물 제작 + 풀 리허설 + 학습

> **정체** — 이 킷은 본선 킷과 **동일한 레일**에 늦지마 콘텐츠(docs/)를 미리 장전하고,
> 운영 모드를 뒤집은 것이다. 본선이 "재생만"이라면 사전과제는 **"학습하며 제작"**이다.
> 목적 3중: ① 사전과제 제출물(~8/10, 톱10 관문) ② 레일 풀 리허설(킷 검증) ③ 학습
> (스킬·캘리브레이션·훅·채점기 제작). **여기서 배운 것이 본선의 탄약이 된다.**

## 1. 본선 모드와의 차이 (이 표가 이 파일의 전부다)

| 축 | 사전과제 모드 (지금) | 본선 모드 (9/4~) |
|---|---|---|
| 주제 | **확정 — docs/ 채워져 있음** (늦지마) | 현장 공개 — 빈 양식 |
| 국면 0 | **생략** — INTENT 검증+Q0만 (컨셉 기확정) | 기획 파이프라인 60분 풀가동 |
| 시계 | 캘린더 이정표(M0~M6, §2) — 국면=순서 | 48h — 국면=시각 |
| 학습 | **ON** — 트래젝토리 로그·복기·스킬 승격·캘리브레이션 | OFF — 재생만 |
| 하네스 | **공사 허용·권장** — 훅 5종·채점기 3종을 여기서 제작·실전 테스트 | 신축 금지 (규칙 1줄 append만) |
| 프롬프트 정제 | 적극 사용 → `prompts/` 라이브러리 축적 | 저장된 prompt_ref 재생 |
| 외부 연결 | **관통 검증이 곧 공정** — CONNECTIONS 체크리스트를 제작 중에 소화 | 검증 완료 전제 |
| eta_min | **실측해서 기록** (스케줄링 근거 생성) | 실측값 사용 |

## 2. 이정표 (M0~M6 — 국면 구조는 본선과 동일, 시각만 캘린더)

| 이정표 | 내용 | 겸사겸사 (학습) |
|---|---|---|
| **M0 관통** | unity-cli 설치·관통(4수) + git·Pages + **빈 WebGL 배포 링크 확인** | CONNECTIONS #1·11·12 · 6.5 호환 판정 |
| **M1 뼈대** | 디렉토리·ARCHITECTURE 골격·스키마·소켓·그레이박스 루프 | 프리커밋 훅·freeze-guard 제작 |
| **M2 양산** | BOM→AAPP 라우팅→에이전트 병렬→소켓 스왑 | Meshy/Trellis/Blender 관통 + **op×모델 캘리브레이션** + 실수→규칙 축적 |
| **M3 분위기** | 조명·오디오·JUICE 레이어 | Suno/Freesound 관통 + 채점기 3종 제작·시험 |
| **M4 동결·문서** | 기능 동결 → 버그 → GDD·AI전략서 최종화 | 라이선스 훅 실전 검증 |
| **M5 제출 (~8/10)** | 링크·영상·문서·소스 점검 → 제출 | 리허설 국면 5 그대로 |
| **M6 사후 복기** | §3 종료 절차 | **킷 v2.0 동결 = 본선 준비 완료 선언** |

## 3. 종료 절차 (M6 — 이걸 해야 리허설이 자산이 된다)

1. 트래젝토리 전체 복기 — 잘 된 것/재시도/실패/낭비 분류 (skill-accumulation §5.2).
2. **캘리브레이션 확정** — op×모델 성공률·실효비용 표를 채워 AAPP 공정 시트의 model/escalation 확정.
3. eta_min 실측값을 표준 공정 시트에 기입 (본선 스케줄링의 근거).
4. 실수→규칙으로 자란 pipelines/ §4를 사람이 검토 — 잘못 굳은 규칙 제거.
5. prompts/ 라이브러리 정리 (중복 통합).
6. **킷 동결** — 이후 본선까지 구조 변경 금지. docs/를 빈 양식으로 되돌린 본선 킷과 교차 확인.

## 4. 주의

- 사전과제 로그는 본선 제출 설계서에 직접 못 쓴다(본선은 본선 기록으로). 단 **"레일을 사전
  검증했다"는 사실과 그 방법론 자체**는 본선 문서·발표의 서사 근거가 된다.
- 학습 모드라도 **동결 규율은 연습한다** — INTENT/규격 동결을 실제로 지키며 제작해야
  본선에서 몸이 기억한다. 어기고 싶어지는 순간이 오면 그게 리허설의 가치다.
- 제출 문서(GDD·AI전략서 HTML)는 킷 밖 산출물 — outputs의 기존 파일을 최종화해 제출.
