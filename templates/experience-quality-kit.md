# 경험 품질 세트 (Experience Quality Kit)
## AI가 놓치는 세 사각지대 — 아트 일관성 · 쥬시 · 전체 경험

> **한 줄 정의** — AI 파이프라인이 구조적으로 놓치는 세 가지(**아트 일관성/가독성 · 쥬시 · 전체 경험**)를 잡는 세트. 셋은 같은 뿌리다 — *명세되지 않고, 개별 검수로 안 잡히고, 전체에서만 보인다.* 그래서 규격으로 명세하고(STYLE·JUICE) 사람이 전체를 보며 검수한다(경험 검수).

- **소속**: `finals-execution-system-v2.md` — STYLE/JUICE는 뼈대(§1.5)에, 경험 검수는 국면 2~4 게이트에
- **구성**: MODULE A `STYLE.md` · MODULE B `JUICE.md` · MODULE C 경험 검수 체크리스트
- **원칙**: 개별 에셋을 완벽히 만들려 하지 말고, **규격 + 후처리 + 전체 검수로 묶는다.**

# MODULE A · STYLE.md (아트 규격 바이블)

> **원리** — AI 에셋은 매 생성이 독립 사건이라 **필연적으로 따로 논다.** 일관성은 기본값이 아니라 강제 산물이다. "예쁘게"(불가) 대신 **측정 가능한 규격**(가능)으로 못 박고, 통일은 생성이 아니라 **후처리**에서 완성한다.

## A1. 팔레트 (고정 색상 + 색의 의미)

모든 에셋·UI·조명이 이 팔레트 안에서만. 색을 고정하면 출처가 달라도 한 세계로 보인다.

```yaml
palette:            # 심야배송 (PS1 야간)
  base_bg:   "#0a0d16"   # 네이비 — 중립 배경
  dominant:  "#ff9f45"   # 나트륨 앰버 — 지배색, 목표·보상
  accent:    "#35e0c8"   # 포스포 시안 — 상호작용 가능
  danger:    "#ff4658"   # 레드 — 위험·실패·부채
color_meaning:      # 색이 의미를 가지면 텍스트 없이 읽힌다
  시안: 상호작용 가능
  레드: 위험 / 실패
  앰버: 목표 / 보상
  네이비: 중립 배경
```

## A2. 밀도 예산 (폴리 · 텍스처)

밀도가 통일돼야 한 스타일로 읽힌다. 하나만 고밀도면 튄다.

| 대상 | 폴리 상한 | 텍스처 |
|---|---|---|
| 소품 | < 1,500 tris | 256px |
| 캐릭터 | < 3,000 tris | 256px |
| 환경 모듈 | < 2,000 tris | 256px |

## A3. 텍스처 · 셰이딩 규약

```yaml
texture:
  resolution: 256px          # PS1 저해상
  sampling: point            # 픽셀 도트감 (안티에일리어싱 없이)
  pbr: false                 # PBR 금지 — 저폴리와 안 맞음
shading:
  model: vertex-lit          # 정점 조명
  dither: true               # 디더링
  fog: true                  # 안개 (거리감 + 디테일 은폐)
```

## A4. 후처리 통일 ⭐ 일관화 치트키

**일관성은 생성이 아니라 후처리에서 완성된다.** 개별 에셋을 안 고치고 **최종 화면에서** 묶는다.

- **팔레트 강제** — 생성 텍스처를 고정 팔레트로 양자화/그레이딩. 색이 제각각 나와도 후처리에서 한 팔레트로 누른다.
- **전체 스타일 셰이더** — 씬 전체에 PS1 포스트프로세싱: 저해상 렌더(320×240 업스케일) · 색 양자화 · 디더링 · 약한 색수차 · 스캔라인. **이 필터 아래 에셋이 뭐든 한 미학으로 묶인다.**
- **조명 통일** — 통일된 조명(사람 수동)이 서로 다른 에셋을 같은 공간감으로 묶는다.

> PS1 스타일 자체가 리스크 완화책이다 — 저해상·저폴리·디더링이 개별 에셋 결함을 **스타일로 흡수**한다.

## A5. 가독성 규약 (일관성 ≠ 가독성)

일관성=한 세계로 보임, 가독성=뭘 할지 보임. 별도로 강제한다.

- **상호작용체 시각 신호** — 집을 수 있는 것·열 수 있는 것에 일관된 신호(미세 시안 외곽선/발광). "이 신호 가진 것만 상호작용"을 학습시킨다.
- **명도 대비 위계** — 목표·경로·상호작용체는 밝게, 배경은 어둡게(야간이라 자연스러움). 눈이 자동으로 중요한 데로 간다.
- **색 코드 일관 적용** — A1의 color_meaning을 어김없이. 시안 보면 만질 수 있다, 레드 보면 위험.

## STYLE.md 빈 양식 (본선용)

```yaml
# ── 상류 승계 (INTENT.md에서 그대로 복사 — 여기서 새로 정하지 않는다) ──
target_emotion: {primary: "", secondary: ""}   # INTENT.aesthetic_primary/secondary (8종 어휘)
genre: ""                                      # INTENT.genre
dimension: <2D | 2.5D | 3D>                    # INTENT.dimension — 룩 결정의 전제
# 이 3칸이 비면 착수 불가. 팔레트·밀도·셰이딩은 전부 target_emotion을 겨냥한 결과여야 한다.

palette: {base_bg: "", dominant: "", accent: "", danger: ""}
color_meaning: {}
density: {prop: "", character: "", env: "", texture: ""}
texture: {resolution: "", sampling: "", pbr: false}
shading: {model: "", dither: true, fog: true}
postprocess: ["전체 스타일 셰이더 명시"]     # 일관화 치트키
readability:
  interactable_signal: ""
  brightness_hierarchy: ""
  color_code: "A1과 일치"
```

# MODULE B · JUICE.md (쥬시 명세)

> **원리** — 쥬시는 명세에 안 적혀서(BOM에 "트럭"은 있어도 "문 닫힐 때 쿵"은 없음), 검증 기준이 없어서(reviewer가 "손맛"을 판정 못 함), 통합의 창발이라(여러 시스템이 동시에 터질 때만 남) **삼중으로 누락된다.** 해법: 쥬시를 **관측 가능한 이벤트에 붙는 반응**으로 명세한다. "재밌게"(불가) → "이 이벤트 → 이 레이어들 동시에"(가능).

## B1. 이벤트–반응 표 (5레이어)

핵심 순간(이벤트)마다 어떤 피드백 레이어가 터지는지 명세. 이러면 에이전트가 실행 가능하다.

**레이어:** 시각 / 청각 / 게임필(진동·펀치) / 시간(히트스톱) / 카메라(셰이크·줌)

| 이벤트 | 시각 | 청각 | 게임필 | 시간 | 카메라 | 크기 |
|---|---|---|---|---|---|---|
| **배송 완료** | 플래시+체크팝+보상숫자 튐 | 딩동+동전 | 스크린펀치(소)·진동(약) | 히트스톱 0.05s | 미세 줌인 | ★큼 |
| **주차 성공** | 주차선 시안 발광 | 브레이크+성공음 | 진동(약) | — | — | 중 |
| **시간초과 실패** | 레드 비네트 펄스 | 낮은 부저 | 강한 진동 1회 | 히트스톱 0.1s | 셰이크(소) | ★큼 |
| **택배 픽업** | 아이템 팝+하이라이트 | 집는 소리 | 펀치(미세) | — | — | 중 |
| **계단 오르기** | 스태미나바 감소 플래시 | 발소리+숨소리 | — | — | — | 소 |

> 작은 순간(계단)엔 1~2레이어, 큰 순간(배송완료·실패)엔 5레이어. **위계가 핵심**(B3).

## B2. ROI 우선순위 (어디에 넣나)

시간 한정 — 아무 데나 넣지 않고 ROI 순서로.

1. **코어 루프 반복 동작** (픽업·배송완료·주차) — 게임 내내 매번 겪음. **최우선.**
2. **성공/실패 순간** — 감정이 튀는 지점. 카타르시스.
3. **영상 10초 장면** — INTENT 핵심 감정 전달. 심사자가 볼 곳.
4. **나머지** — 시간 남으면.

## B3. 과잉 방지 (쥬시도 독이 된다)

모든 게 번쩍이면 아무것도 안 도드라진다(감각 과부하 = 싸구려).

- **조용한 기본 상태 유지** — 강조는 대비에서 나온다. 평상시가 잔잔해야 쥬시가 터질 때 산다.
- **순간별 레이어 위계** — 큰 순간 5레이어, 작은 순간 1~2. 다 똑같이 강하면 위계가 죽는다.
- **한 순간에 다 쓰지 말 것** — 잘 orchestrated된 한 순간 > 산발적 마이크로 인터랙션 다수.

## JUICE.md 빈 양식 (본선용)

```yaml
target_emotion: {primary: "", secondary: ""}   # INTENT.aesthetic_primary/secondary 승계
events:
  - name: ""
    visual: ""
    audio: ""
    gamefeel: ""      # 진동·펀치
    time: ""          # 히트스톱
    camera: ""        # 셰이크·줌
    size: 큼|중|소     # 레이어 수 결정
    serves: ""        # 이 반응이 target_emotion에 기여하는 방식 한 줄.
                      # 못 쓰겠으면 그 쥬시는 장식이다 — 빼거나 size를 낮춘다
roi_priority: [코어루프_반복, 성공실패, 영상장면, 나머지]
anti_overkill: [조용한_기본상태, 순간별_위계, 한순간_집중]
```

> **감정 겨냥 검사 (B1 작성 후 1분)** — 같은 이벤트 표라도 목표 감정에 따라 답이 달라진다.
> `도전`이면 성공/실패의 대비를 키우고, `발견`이면 새로운 것이 드러나는 순간에 레이어를 몰고,
> `감각`이면 반복 동작의 질감에 투자하고, `서사`면 정보가 밝혀지는 컷에 시간(히트스톱)을 쓴다.
> **size가 ★큼인 이벤트 전부가 `serves`를 못 채우면 쥬시 배분이 감정과 어긋난 것이다.**

# MODULE C · 경험 검수 체크리스트 (사람 게이트)

> **원리** — 아트 일관성·가독성·쥬시는 **개별 에셋 검수로 안 잡힌다.** reviewer는 "폴리<1500"은 검증해도 "옆 에셋과 톤 맞나"는 판정 못 한다. 전체에서만 보이므로 **사람이 소유하는 게이트**. 자동화 불가.

**언제:** 국면 2 통합 체크포인트마다(4h) + 국면 3~4 플레이스루. **자주** 해야 열 개 만든 뒤 다 뜯는 참사를 막는다.
**방법:** 씬 전체 스크린샷 + 실제 플레이스루.

### C1. 아트 일관성
- ☐ 씬 스크린샷이 **한 세계**로 보이나? 튀는 에셋 없나?
- ☐ 폴리·텍스처 밀도가 균일한가? (하나만 고밀도로 튀지 않나)
- ☐ 팔레트(A1) 벗어난 색이 없나?
- ☐ 조명이 통일돼 있나?

### C2. 가독성
- ☐ **3초 안에** 뭘 해야 할지 읽히나?
- ☐ 상호작용체가 시각적으로 구분되나? (A5 신호)
- ☐ 목표·경로가 눈에 먼저 들어오나? (명도 위계)

### C3. 쥬시
- ☐ 코어 루프 반복 동작에 피드백이 있나? (B2 최우선)
- ☐ 성공/실패가 감각적으로 확실히 구분되나?
- ☐ 과잉으로 싸구려가 되진 않았나? (B3)

### C4. AI 사각지대 (쥬시 너머)
- ☐ **온보딩** — 첫 30초에 플레이어가 뭘 할지 아나? (AI는 "처음 만지는 혼란"을 상상 안 함)
- ☐ **완급** — 계속 긴장만이 아니라 쉬어가는 리듬이 있나?
- ☐ **톤 드리프트** — INTENT 톤에서 미묘하게 벗어난 곳 없나?
- ☐ **부정 공간** — 과하게 채우지 않았나? (AI는 채우는 건 잘해도 비우는 걸 못 함)

**판정:** 하나라도 ✗면 해당 모듈(STYLE/JUICE)로 돌아가 수정. 통합 체크포인트마다 반복.

# 시스템 통합 — 어디에 앉나

| 모듈 | 위치 | 시간대 |
|---|---|---|
| **STYLE.md** | 뼈대(§1.5)에 박음 — 모든 에셋 생성 프롬프트가 참조 | 레일 (준비) + 국면 1 확정 |
| **JUICE.md** | 뼈대에서 핵심 이벤트 정의 → 국면 3에서 레이어 적용 | 국면 1 정의 + 국면 3 실행 |
| **경험 검수** | 국면 2 통합 체크포인트(4h) + 국면 3~4 플레이스루 | 사람 게이트, 상시 |

**핵심 흐름:** STYLE 규격이 생성을 묶고 → 후처리(전체 셰이더)가 일관성을 완성하고 → JUICE가 반복 순간을 살리고 → 경험 검수가 전체에서만 보이는 것을 사람이 잡는다.

> STYLE·JUICE는 §12 변경관리의 동결 대상(국면 1 확정 후 수정 잠금, 직교 확장만). 경험 검수는 reviewer(자동)가 못 잡는 것을 사람이 소유하는 게이트로, "AI 안 쓰는 구간"에 정식 편입된다.
