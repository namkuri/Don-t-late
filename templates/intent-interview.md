# INTENT 인터뷰 질문지
## 의도를 30분 안에 애매하지 않게 뽑는 도구

> **한 줄 정의** — 주제 공개 직후, 이 질문지를 읽으며 **객관식으로 답만 고르면** `INTENT.md`가 완성된다. 자유 서술 금지 — 강제 선택이 속도·명확성·불변성(계약)을 동시에 만든다.

- **소속**: `finals-execution-system-v2.md` 국면 0 (T+0~0.5h)의 실행 도구
- **원리**: 좋은 질문 = ① 하류를 결정하는 것만 · ② 상호배타적으로 갈리게 · ③ 관측 가능하게
- **사용법**: 순서대로 답한다(추상→구체). 각 답이 앞 답과 **모순되면 즉시 고친다.**

## PART 1 · 불변축 (가장 안 바뀔 것부터)

### Q1 · 톤 ⭐ 최우선
**이 게임의 지배적 톤은?**
- ☐ A. 다크코미디 (웃프게)
- ☐ B. 진지·몰입
- ☐ C. 부조리·블랙유머
- ☐ D. 서정·잔잔

→ **결정하는 것:** 모든 아트·콘텐츠·사운드의 방향. 한 번 정하면 안 바꾼다.
→ **심야배송 예:** `A. 다크코미디`

### Q2 · 핵심 감정 (= 선택 카드의 **A 에스테틱** 승계)
**영상 10초에 심사자가 느껴야 할 단 하나의 감정은?**

> **원칙: 새로 고르지 않는다.** P2에서 선택된 카드의 `A 에스테틱`을 그대로 가져온다.
> 여기서 다시 고르면 기획 발산·채점이 무의미해진다. 바꾸려면 카드 선택을 되돌려라.

MDA 8종 어휘 (planning.md §2 P1과 동일 체계 — 주 감정 1 + 보조 1까지):
- ☐ 감각 (sensation — 시청각적 쾌감·질감)
- ☐ 판타지 (fantasy — 다른 존재가 된 기분)
- ☐ 서사 (narrative — 이야기가 펼쳐지는 감각)
- ☐ 도전 (challenge — 장애물 극복·유능감)
- ☐ 교우 (fellowship — 사회적 유대) ※ 1인 48h·싱글에선 통상 배제
- ☐ 발견 (discovery — 미지를 밝히는 쾌감)
- ☐ 표현 (expression — 자기를 드러냄·흔적 남기기)
- ☐ 복종 (submission — 리듬에 몸을 맡기는 몰입)

→ **결정하는 것:** 코어 루프·비주얼의 판정 기준. "이 에셋이 이 감정에 기여하나?"
→ **하류 계약:** 이 값이 `STYLE.md`의 `target_emotion`과 `JUICE.md`의 겨냥 대상이 된다.
   여기가 비면 experience-quality-kit이 착수 불가다.
→ **심야배송 예:** `주=서사, 보조=감각` (씁쓸한 공감 = 서사 계열)

### Q3 · 우선순위
**48h 리소스가 부족할 때 무엇을 지키나?**
- ☐ A. 비주얼 임팩트 우선 (첫인상·각인)
- ☐ B. 플레이 깊이 우선 (재미·시스템)

→ **결정하는 것:** 자원 배분. 무엇을 포기할지. (둘 다 못 가진다)
→ **심야배송 예:** `A. 비주얼 임팩트 우선` (PS1 야간 감성으로 각인)

### Q4 · 플레이어 판타지
**플레이어는 무엇이 된 기분인가?**
- ☐ A. 쫓기는 밑바닥 노동자
- ☐ B. 유능한 프로
- ☐ C. 무법 질주자
- ☐ D. 관찰자·수집가

→ **결정하는 것:** 코어 루프의 동사(무엇을 반복하나).
→ **심야배송 예:** `A. 쫓기는 밑바닥 노동자` (빚 갚는 야간 배달)

## PART 2 · 범위 (must / never는 짝)

### Q5 · must_have
**이것 없으면 이 게임이 아니다 — 1~2개만.** (많으면 스코프 초과)
- 예시 후보: 코어 루프 / 특정 분위기 / 특정 메카닉 / 특정 감정 순간

→ **결정하는 것:** 절대 안 자를 것. 시간 부족해도 이건 지킨다.
→ **심야배송 예:** `① 배달 코어루프(픽업→주행→주차→운반→보상)` `② PS1 야간 밤골목 분위기`

### Q6 · never ⭐ 스코프 방어선
**절대 하지 않을 것 — 목록.** (나중에 "추가할까?" 유혹의 방어선)
- 흔한 후보: 멀티플레이어 / 오픈월드 / 커스텀 캐릭터 애니 대량 / 실제 상표명 / 다중 레벨

→ **결정하는 것:** 스코프 크립 차단. 변경관리의 "확장 허용" 판정 기준.
→ **심야배송 예:** `멀티플레이어 · 오픈월드 · 커스텀 애니 대량 · 실제 브랜드명(쿠팡→번개배송) · 다중 씬`

> **핵심** — must와 never 사이가 곧 스코프다. never를 명확히 뽑을수록 나중에 안 흔들린다.

## PART 3 · 구체

### Q7 · AI 공정 레버리지 ⭐ 심사 축 (구 "AI 쇼케이스 훅")
**"AI 도구로 이 게임을 어떻게 만들었나" 최소 2개.** (없으면 이 대회에서 진다)

> **⚠⚠ 여기가 런타임 AI 오독을 잡는 유일한 관문이다.**
> 컨셉 카드에는 AI 칸이 없다(기획 단계는 재미만 판단). 즉 이 질문 전까지 오독은 무검출로 생존한다.
> **런타임 AI가 스코프에 섞여 있으면 여기서 반려하고 빌드타임 생성으로 강등한다.**
> 강등이 불가능한 컨셉이면 그건 이 대회에 맞지 않는 컨셉이다 — P2로 되돌린다.
>
> **⚠ 이것은 개발 공정에 대한 질문이다. 게임의 기능이 아니다.**
> 심사 축은 **빌드타임** — "무엇을 만들었나"가 아니라 "AI를 어떻게 부렸나".
> **런타임 AI(플레이 중 LLM 호출·AI NPC)는 심사 축이 아니며, 기본 금지다** —
> WebGL 제출물에서 API 키 노출·비용·오프라인 실패를 떠안는다.
> 판별법: **생성물이 빌드에 구워지는가?** 구워지면 ✅, 플레이 중 호출하면 ❌.

- ☐ 소품·에셋 배치 생성 (Meshy/Trellis → 임포트)
- ☐ 로직 자동화 (Claude Code + unity-cli로 코드 작성·검증)
- ☐ 콘텐츠 데이터 생성 (대사·아이템·레벨을 LLM으로 생성 → **CSV/JSON으로 구워 넣음**)
- ☐ 검수 자동화 (reviewer 에이전트가 규격·데이터 정합성 판정)
- ☐ 기타: __________

→ **결정하는 것:** 심사 축 강제. **컨셉을 비틀 필요는 없다** — 어떤 게임이든 AI 공정으로 만들 수 있다.
   비틀 게 아니라, 이 컨셉의 **어디에 공정을 붙일지**를 고르는 칸이다.
→ **심야배송 예:** `① 소품 배치 생성` `② 배송메모·후기 LLM 생성` (+로직 자동화)

### Q8 · 비교 좌표
**무엇처럼 / 무엇과 다르게?** (기존 게임에 앵커링)
- 형식: "___의 [요소]를 가져오되, ___와 달리 ___"

→ **결정하는 것:** 방향을 구체 좌표로 고정. 에이전트가 참조할 레퍼런스.
→ **심야배송 예:** `"Death Stranding의 배달 긴장 + PS1 저해상 + 한국 밤골목. 단, 진지한 SF가 아니라 생활밀착 다크코미디."`

---

### Q9 · 변별성 (질문형 ⭐ 상대평가 대비)
**오케스트레이터에게 묻는다: "이 주제로 다른 팀들이 낼 뻔한 접근 3개를 예상해라."**
- 우리 스코프가 그중 하나와 겹치나? → 겹치면 **비틀 지점 1개** 지정 (스코프 유지 범위 내)
→ **결정하는 것:** 열 팀이 같은 계산을 하면 같은 게임이 나온다. 실현성 게이트만으론 변별이 안 된다.
→ **심야배송 예:** `뻔한 접근: 일반 배달 시뮬 → 비틀기: 빚·야간·다크코미디 생활밀착`

## 진행 중 모순 검문 (인터뷰 도중 즉시)

각 답을 앞 답과 대조. 충돌하면 그 자리에서 하나를 고친다.

| 충돌 예 | 판정 |
|---|---|
| 톤=다크코미디 + 핵심감정=따뜻한 감동 | ✗ 충돌 → 감정을 "웃픔"으로 |
| 우선순위=플레이 깊이 + must=화려한 비주얼 | ✗ 충돌 → 하나 포기 |
| never=커스텀 애니 + 판타지=화려한 액션 | ✗ 충돌 → 판타지 조정 |

## 마감 관문 · 실행가능성 테스트

인터뷰 종료 전 마지막 5분. 이 테스트를 통과할 때까지 끝내지 않는다.

> **"이 INTENT.md만 주고 낯선 개발자에게 '이 게임 만들어'라고 하면, 되묻지 않고 시작할 수 있는가?"**

체크:
- ☐ 톤을 되묻지 않는가? (Q1이 명확한가)
- ☐ **장르를 되묻지 않는가?** (`genre`·`dimension`·`camera` 3칸이 채워졌는가 —
      비면 TECH_SPEC이 카메라·스케일·콜리전을 처음부터 다시 묻는다)
- ☐ **목표 감정이 8종 어휘로 지목됐는가?** (`aesthetic_primary` —
      비면 STYLE·JUICE가 겨냥할 표적이 없다)
- ☐ 뭘 우선할지 되묻지 않는가? (Q3이 갈렸는가)
- ☐ 스코프 경계가 명확한가? (must/never가 채워졌는가)
- ☐ AI 공정 레버리지가 2개 이상 박혔는가? (Q7)
- ☐ **런타임 AI 호출이 스코프에 없는가?** (있으면 빌드타임 생성으로 강등하거나 `never`에 넣는다)
- ☐ 앞뒤 답에 모순이 없는가? (특히 `aesthetic_primary`가 선택 카드의 A와 같은가 —
      다르면 기획 결과를 덮어쓴 것이다)

하나라도 ✗면 그 질문으로 돌아간다.

**동결 직전 마지막 절차 — 반론 게이트 (Q0, 5분):**
오케스트레이터에게 묻는다: *"이 INTENT/스코프에서 뭐가 빠졌나? 반론이 있다면? 심사자 관점 약점 3개는?"*
(응답 규율·반영 기준은 `question-gates.md` §0 — 아첨 금지, 반영은 치명·심사직결·5분미만만, 기각은 기록)

반론 처리까지 끝나면 **INTENT 동결(freeze) — 이후 국면에서 수정 금지, 직교 확장만 허용.**
실행가능성 테스트는 *명확성*을, 반론 게이트는 *타당성*을 검사한다 — 명확하게 틀린 방향도 전자는 통과시킨다.
