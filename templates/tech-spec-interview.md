# TECH_SPEC 인터뷰 + 규격 양식
## 게임의 "규격(계약)"을 뼈대에서 뽑는 도구

> **한 줄 정의** — STYLE.md가 "룩(색·폴리·텍스처)"이라면 TECH_SPEC.md는 **"규격(스케일·리깅·카메라·물리)"**이다. 이것들은 에셋·시스템 간 **인터페이스 계약**이라, 뼈대에 명시하지 않으면 연쇄로 다 틀어진다(캐릭터가 문을 못 지나가고, 애니가 깨지고, 손이 물건을 못 잡는다).

- **소속**: `finals-execution-system-v2.md` 뼈대(§1.5) — STYLE 인터뷰와 나란히
- **원리**: 규격은 두 층 — **계약(일찍 동결)** vs **튜닝(늦게까지 유동)**
- **사용법**: PART 1(계약) 먼저 확정·동결 → PART 2(애니) → PART 3(튜닝)은 기본값만

---

## 0. 두 층 원리 (가장 중요)

| 층 | 성격 | 동결 시점 | 예 |
|---|---|---|---|
| **계약 (Contract)** | 틀리면 **부서짐**. 에셋·시스템 인터페이스 | 뼈대(국면1) **동결** | 단위·키·비율·리그·원점·좌표·콜리전·카메라 |
| **튜닝 (Tuning)** | 감으로 **조정**. 플레이하며 맞춤 | 국면 3~4까지 **유동** | 이동속도·카메라거리·난이도·믹스·쥬시타이밍 |

> 계약은 ARCHITECTURE처럼 일찍 얼리고, 튜닝은 에셋처럼 늦게까지 연다(§12 동결 캐스케이드와 동일).

## 도미노 원리 ⭐

**캐릭터 기준 키(단일 숫자)가 첫 조각.** 그게 정해지면 문·계단·천장·차량·카메라 거리·레벨 스케일이 줄줄이 결정된다. **이걸 제일 먼저 박는다.**

```
캐릭터 키 1.8u → 문 2.1u → 한 층 3u → 계단 참 → 차량 → 카메라 거리 → 레벨 전체
```

## PART 1 · 계약 규격 (뼈대에서 동결)

### Q0 · 장르 승계 ⭐ 도미노보다 앞선 전제 (묻지 말고 받아적는다)
`docs/INTENT.md`에서 **그대로 복사한다.** 여기서 새로 정하면 기획 결정을 덮어쓰는 것이다.

- `genre`: ______________ (예: 3D 1인칭 공포)
- `dimension`: ☐ 2D · ☐ 2.5D · ☐ 3D
- `camera`: ☐ 1인칭 · ☐ 3인칭 추적 · ☐ 탑뷰 · ☐ 사이드뷰

→ **결정하는 것:** 이 인터뷰의 **적용 범위 자체.**
→ **INTENT에 이 값이 없으면 인터뷰를 시작하지 마라** — 국면 0으로 돌려보낸다.
→ **차원별 분기 주의:**
  - `2D` 선택 시 — Q3(신체 비율)·Q4(업축)·Q6(리그)는 **해당 없음**으로 건너뛰고,
    대신 픽셀 퍼 유닛(PPU)·스프라이트 피봇·정렬 축(sorting axis)을 규격에 추가한다.
    아래 Q1·Q2·Q5·Q7은 3D를 암묵 전제하므로 2D 어휘로 바꿔 읽어라.
  - `2.5D` 선택 시 — 3D 에셋 + 고정 시점. Q1~Q7 그대로 쓰되 카메라 각도를 Q8에 명시.
→ **심야배송:** `3D / 3인칭 추적`

### Q1 · 기준 단위
- ☐ 1 유닛 = 1 미터 (권장 기본값)
- ☐ 커스텀: ______
→ **심야배송:** `1u = 1m`

### Q2 · 캐릭터 기준 키 ⭐ 도미노 첫 조각
**주인공 키 = ___ 유닛** (모든 스케일의 기준)
→ **심야배송:** `1.8u`

### Q3 · 신체 비율
- ☐ 리얼 (7~8등신)
- ☐ 중간 (5~6등신)
- ☐ 데포르메 (2~3등신 SD)
→ **결정:** 전 캐릭터 통일 기준 · **심야배송:** `중간 (PS1 감성)`

### Q4 · 좌표 · 업축 규약
- ☐ Y-up, +Z 정면 (Unity 기본, 권장)
- ☐ 커스텀
→ **결정:** 임포트 시 회전 안 틀어지게 · **심야배송:** `Y-up, +Z front`

### Q5 · 원점 / 피봇 규약 ⭐ 소켓 스왑의 지뢰
- ☐ 프롭 = 바닥 중심 / 캐릭터 = 발 중심 (권장)
- ☐ 커스텀
→ **결정:** 배치 정확도 · **심야배송:** `프롭=바닥중심, 캐릭터=발중심`

### Q6 · 리그 표준
- ☐ Mixamo 호환 humanoid (애니 리타깃 가능, 권장)
- ☐ 커스텀 리그
- ☐ 없음 (정적 캐릭터)
→ **결정:** 애니 재사용 가능성 · **심야배송:** `Mixamo humanoid`

### Q7 · 콜리전 레이어 표준
- ☐ Player / Static / Interactable / Trigger (권장 4종)
- ☐ 커스텀
→ **결정:** 에이전트가 콜라이더 붙일 때 참조 · **심야배송:** `4종 표준`

### Q8 · 카메라 파라미터 (시점은 Q0에서 이미 확정됨)
> 시점을 **다시 고르지 않는다** — Q0의 `camera`가 답이다. 여기서는 그 시점의 수치만 정한다.

- FOV: ______ · 거리/높이(3인칭·탑뷰): ______ · 피치 각: ______
- 추적 지연(damping): ______ · 충돌 회피 여부: ☐
→ **결정:** 레벨 스케일·가독성·조작 · **심야배송:** `3인칭 추적, FOV 60, 거리 5u, 피치 20°`

### Q9 · WebGL 성능 예산 (씬 전체)
- 총 폴리 상한: ______ · 드로우콜 상한: ______ · 텍스처 메모리: ______
→ **결정:** 막판 빌드 붕괴 방지 · **심야배송(예시):** `총 <150k tris, 드로우콜 <200, 텍스처 <128MB`

### Q9' · 자기완결 빌드 확인 ⭐ 제출물 안전핀
- ☐ **런타임 외부 네트워크 호출 없음** (LLM API·원격 데이터·외부 CDN 전부 없음)
- ☐ 모든 콘텐츠가 빌드에 포함됨 (생성 데이터는 CSV/JSON으로 구워짐)
- ☐ API 키·시크릿이 빌드 산출물에 없음

→ **결정하는 것:** 제출물이 **오프라인·무인 환경에서도 그대로 도는가.**
   WebGL 빌드는 클라이언트에 전부 노출된다 — 키를 넣으면 공개된다.
   런타임 호출은 CORS·비용·서비스 장애까지 제출물의 리스크로 떠안는 일이다.
→ **예외가 필요하면 사람 승인 게이트.** 기본값은 "없음".

## PART 2 · 애니메이션 규격 (계약성)

### Q10 · 애니 최소 세트
**필요한 애니 목록** (적을수록 좋음)
→ **심야배송:** `idle · walk · run · 짐들기 · 짐놓기` (5종, Mixamo 리타깃)

### Q11 · 루트 모션 vs 인플레이스
- ☐ 인플레이스 (코드가 이동, 권장 — 제어 쉬움)
- ☐ 루트 모션 (애니가 이동)
→ **결정:** 이동 시스템 구조 · **심야배송:** `인플레이스`

### Q12 · IK 적용 ⭐ 당신이 짚은 것
- ☐ 손 IK (물건 잡기 정확도) — 쓸까?
- ☐ 발 IK (계단·경사 지면 접지) — 쓸까?
- ☐ 안 씀 (비용 절약)
→ **결정:** 손맛 vs 구현비용 · **심야배송:** `손 IK O(택배 잡기 가치 큼), 발 IK △(계단 많지만 비싸면 생략)`

## PART 3 · 튜닝 규격 (기본값만 · 국면3~4 조정)

> 여기는 확정이 아니라 **시작 기본값.** 플레이하며 사람이 감으로 맞춘다(AI 안 씀 구간).

### Q13 · 이동 속도 (기본값)
걷기 ___ u/s · 뛰기 ___ u/s → **심야배송:** `걷기 2.5, 뛰기 5`

### Q14 · 카메라 세팅 (기본값)
거리 ___ u · 높이 ___ u · FOV ___° → **심야배송:** `거리 4, 높이 2, FOV 60`

### Q15 · 프레임 타겟
- ☐ 60fps · ☐ 30fps → **결정:** 히트스톱(프레임)·물리 스텝 · **심야배송:** `60fps` (JUICE 타이밍 초 단위와 연동)

### Q16 · 오디오
3D 사운드: ☐ O ☐ X · 믹스 비율(BGM:SFX:앰비)
→ **심야배송:** `3D 사운드 O, 믹스 6:8:4`

### Q17 · UI
다이제틱: ☐ O(게임 내 폰) ☐ X(HUD) · 폰트(한글 지원 필수)
→ **심야배송:** `다이제틱 폰 O, 비트맵 한글 폰트`

### Q18 · 인게임 시간 (해당 시)
게임 내 시계? 게임 1분 = 실제 ___초
→ **심야배송:** `있음, 게임 1분 = 실제 4초` (야간 시간제한 메카닉)

### Q19 · 난이도 파라미터 (사람 튜닝)
시간제한·스태미나 소모율 등 — 플레이하며 조정
→ **심야배송:** `국면3에서 플레이 감각으로 조정`

## 출력 A · TECH_SPEC.md (심야배송 예시)

```yaml
# === 계약 (동결) ===
unit: "1u = 1m"
character_height: 1.8            # 도미노 첫 조각
body_ratio: 중간_5-6등신
coordinate: "Y-up, +Z front"
pivot: {prop: 바닥중심, character: 발중심}
rig: Mixamo_humanoid
collision_layers: [Player, Static, Interactable, Trigger]
camera_type: 3인칭_추적
webgl_budget: {tris: 150000, drawcalls: 200, texture_mb: 128}

# === 애니 ===
anim_set: [idle, walk, run, pickup, putdown]
locomotion: 인플레이스
ik: {hand: true, foot: false}   # 손 IK만 (택배 잡기)

# === 튜닝 (기본값, 국면3~4 조정) ===
move_speed: {walk: 2.5, run: 5.0}
camera: {dist: 4, height: 2, fov: 60}
frame_target: 60
audio: {spatial: true, mix: {bgm: 6, sfx: 8, amb: 4}}
ui: {diegetic: true, font: "bitmap_korean"}
ingame_time: {enabled: true, game_min_per_real_sec: 4}
difficulty: "국면3 플레이 튜닝"
```

## 스케일 레퍼런스 시트 ⭐

추상 숫자 대신 **캐릭터 대비 상대 척도**로 모든 에셋을 만든다. 이 시트를 에셋 프롬프트에 첨부.

| 오브젝트 | 크기 (캐릭터 키 1.8u 기준) |
|---|---|
| 주인공 | 1.8u (기준) |
| 문 | 2.1u 높이 |
| 한 층 높이 | 3.0u |
| 계단 참 | 2.8u 폭 |
| 포터 트럭 | 4.5u 길이 · 2.0u 높이 |
| 택배 상자(중) | 0.4u |
| 엘리베이터 | 1.5×1.5×2.2u |

> 모든 에셋이 캐릭터 대비로 만들어지면 스케일이 안 틀어진다. 이게 계약 규격의 실전 형태.

## 마감 관문

- ☐ 캐릭터 키(도미노 첫 조각)가 박혔나?
- ☐ 계약 규격 9개가 다 채워졌나? (튜닝은 기본값이면 OK)
- ☐ 원점·좌표·콜리전이 명시됐나? (소켓 스왑·상호작용의 전제)
- ☐ 스케일 레퍼런스 시트가 나왔나?
- ☐ WebGL 예산이 정해졌나?

전부 ✓면 **계약 규격 동결** — 국면 1 이후 수정 금지(직교 확장만). 튜닝 규격은 국면 3~4까지 유동.
