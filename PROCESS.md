# PROCESS.md — 프로젝트 진행 명세 (관제용 상태 기계)

> 오케스트레이터가 **현재 위치·다음 결정·병목**을 판정하는 단일 기준.
> 사람은 이 파일을 읽을 필요 없다 — 오케스트레이터가 이걸 대조해 브리핑한다.
> (사람용 검증 창구: 브리핑의 `근거: PROCESS §N` 포인터로 역추적 가능)

## 상태 저장 (STATUS.md — T+0에 오케스트레이터가 생성, planning/에)
```yaml
t0: "YYYY-MM-DD HH:MM"        # 본선 시작 실측 시각 (사람이 확정)
phase: 0                       # 0~5
stage: "P0"                    # 국면 내 스테이지
pending_decisions: []          # 사람 결정 대기 목록
blocked: []                    # CONNECT_REQUEST 등
last_checkpoint: null          # 마지막 4h 체크포인트
clock_note: ""                 # 예산 대비 (정상|지연 N분|여유)
```
- 시간 계산: `date` 실행값 − t0 = T+경과. 국면 예산과 대조.
- 상태 전이는 **전환 조건 충족 + (사람 결정 필요 시) 승인** 후에만.

## 병목 감지 규칙 (오케스트레이터 상시 대조)
| 신호 | 판정 | 조치 제안 |
|---|---|---|
| review 상태 ≥3 적체 | 검증 병목 | reviewer 병렬 추가, 생성 투입 감속(TOC) |
| doing 항목 경과 > eta_min×2 | 정체 | 상태 확인 → 재시도/에스컬레이션 |
| blocked에 CONNECT_REQUEST 미해결 | 레인 정지 | 사람에게 재상기 (긴급도 순) |
| T+경과 > 국면 예산 | 일정 지연 | sacrifice 집행 제안 (Q2' 연동) |
| pending_decisions ≥1 | **사람이 병목** | 브리핑 최상단에 결정 푸시 |
| WIP > 5 | 재공품 초과 | 신규 착수 정지, 검수·통합 우선 |
