# 파이프라인: 캐릭터·애니 (character-anim)
담당: asset-generator(모델) + **사람 매개 구간** + unity-dev(임포트) · 연결: #2 Meshy · #6 Mixamo 🖐 · #7 Cascadeur 🖐
⚠ 최장 공정(LPT) — **국면 2 최선두 발사.** 늦게 시작하면 통합에 못 맞춘다.

## 1. 기동 점검
- [ ] Meshy 확인 — 미연결 → CONNECT_REQUEST(치명)
- [ ] Mixamo 절차서(준비기간 작성) 존재 확인 — 없으면 사람에게 절차 확인 요청
- [ ] TECH_SPEC: 리그 표준(Mixamo humanoid)·신체비율·기준 키·애니 최소 세트 확인

## 2. 공정 단계 (🖐 = 사람 손 필요 — 에이전트는 준비+핸드오프까지)
1. 모델 생성 — 신체비율·키·T-pose 명시. 실루엣 판독 가능 여부가 1차 기준
2. 🖐 **Mixamo 핸드오프**: 에이전트가 FBX 정리·업로드 체크리스트 준비 → 사람에게
   `[HANDOFF] 파일·할 일(업로드→자동리깅→애니 N종 다운로드)·반환 경로` 형식으로 요청
   — 반환 경로는 `assets/_intake/mixamo/` 고정 (사람 매개 반입물도 같은 관문, HARNESS §9)
3. 리타깃 검증 — 본 네이밍·T-pose·애니 세트 재생 확인 (unity-dev)
4. 🖐 웨이트 깨짐은 육안 판정(사람) — 깨짐 발견 시 Blender 수동 보정 or Cascadeur
5. 임포트·소켓 스왑

## 3. 자동교정 루프
| 게이트 실패 | 자가 교정 (cap 2) |
|---|---|
| T-pose 아님 | 생성 프롬프트에 "T-pose, arms horizontal" 강화 재생성 |
| 본 네이밍 불일치 | 리네임 매핑 시도 → 실패 시 사람 |
| 애니 재생 깨짐 | 임포트 설정(휴머노이드 지정) 재시도 |
- 웨이트·실루엣 등 육안 판정은 자가교정 대상 아님 → 바로 사람.

- [시드] 리깅 후 비율 수정 요구 → 비율·키는 생성 전 TECH_SPEC 확정값만 사용
