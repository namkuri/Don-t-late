# 파이프라인: 빌드·배포 (build-deploy)
담당: unity-dev + orchestrator (게이트: 사람) · 연결: #1 Unity MCP · #11 git/GitHub · #12 GitHub Pages
원칙: **상시 트랙.** T+2h 빈 씬 관통이 의무. 링크는 48h 내내 살아 있어야 한다.

## 1. 기동 점검
- [ ] Unity MCP(빌드 실행 가능) — 미연결 → CONNECT_REQUEST(치명)
- [ ] git 원격 + 인증 확인 — 미연결 → CONNECT_REQUEST(치명 — 소스 제출 규정 직결)
- [ ] Pages 배포 경로 확인 — 미연결 → CONNECT_REQUEST(높음, 폴백: 대안 정적 호스팅)

## 2. 공정 단계
1. **T+2h 관통**: 빈 씬 WebGL 빌드 → 배포 → 다른 브라우저에서 링크 열림 확인
2. 이후: 코어루프·통합 체크포인트마다 재빌드·재배포
3. 커밋: 자주, `(generated via X) [reviewed]` 태그, 미검증물 금지
4. T-3h: 기능 동결 후 최종 빌드 → 국면 5 리허설로

## 3. 자동교정 루프 (cap 2)
```
빌드 → 실패? → 로그 읽기 → 원인 수정 → 재빌드            (cap 2 → 사람)
배포 → 링크 HTTP 200 + 로드 확인 → 실패? → 배포 설정 점검 → 재배포
```
- 배포 성공은 조용히(`링크 OK` 1줄), 실패는 시끄럽게(로그 요약+가설).
- **프리커밋 훅(레일)**: ① 컴파일 실패 커밋 차단 ② 동결 파일 수정 차단(freeze-guard, §12 연동)
  ③ 커밋 태그 누락 경고. 훅이 막으면 고칠 때까지 커밋이 안 되는 구조다.

## 4. 실수 → 규칙 (append-only)
- [시드] 빌드 확인을 통합 후로 미룸 → 통합 즉시 재빌드 (씬은 항상 플레이 가능)
- [시드] WebGL 비호환 기능(스레드·파일IO) 도입 후 막판 발견 → 도입 전 플랫폼 제약 확인
- [시드] 링크 확인을 같은 브라우저 캐시로만 함 → 반드시 다른 브라우저/시크릿 창
- [시드] 런타임 외부 API 호출(LLM·원격 데이터) 도입 → 키 노출·CORS 차단·오프라인 실패.
  **WebGL 제출물은 자기완결 빌드**여야 한다. 생성 콘텐츠는 빌드타임에 구워 넣는다
