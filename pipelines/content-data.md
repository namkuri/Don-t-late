# 파이프라인: 데이터 콘텐츠 (content-data)
담당: orchestrator 직접 또는 위임 (검증: reviewer) · 연결: #13 스키마 검증 스크립트 (내장 — 외부 불필요)
용도: 텍스트·이벤트·아이템 등 LLM 배치 생성 콘텐츠. **AI 공정 레버리지 축 — 거의 공짜 물량.**

> **⚠ 이 파이프라인의 생성은 전부 빌드타임이다.** LLM은 **개발 중에** 콘텐츠를 찍어내고,
> 결과물은 CSV/JSON으로 **구워져 빌드에 포함된다.** 플레이 중 LLM을 호출하지 않는다.
> 다양성이 필요하면 **미리 대량 생성해두고 런타임엔 셔플·선택만** 한다.
> 런타임 외부 호출은 심사 축이 아닐뿐더러 WebGL에서 API 키 노출·CORS·비용·오프라인 실패를 부른다.

## 1. 기동 점검
- [ ] data/schemas/ 에 대상 스키마 존재 — 없으면 생성 거부(뼈대 미완성, 스키마부터)
- [ ] scripts/validate_schema.py 존재 — 없으면 CONNECT_REQUEST(보통, 폴백: 수동 검토)
- [ ] INTENT.md의 tone·never 목록 로드 (금지어 필터 기준)

## 2. 공정 단계
1. 생성 — "오직 JSON, 서두·설명·마크다운 금지, 스키마 정확 준수" 명시. 수량·길이 규격 포함
2. 스키마 검증 스크립트 실행
3. 중복·금지어 검사
4. data/content/ 저장 + 건수 기록
5. **빌드 반영 확인 ⭐ 생략 금지** — 생성물을 Unity가 실제로 읽는 경로에 배치하고
   (`Assets/` 하위 · Resources/Addressable/StreamingAssets 중 택1), 리포지토리에 커밋한다.
   **"생성했다"가 아니라 "빌드에 들어갔다"가 done이다.** 런타임 외부 호출로 대체하지 않는다.

## 3. 자동교정 루프 (cap 2)
| 게이트 실패 | 자가 교정 |
|---|---|
| JSON 파싱 실패 | 서두/백틱 제거 재시도 → 재생성(출력 형식 지시 강화) |
| 스키마 위반 | 위반 필드만 재생성 (전체 재생성 금지 — 낭비) |
| 중복 | 중복 제거 → 부족분만 추가 생성 |
| 금지어(never) 포함 | 해당 항목만 교체 생성 |
| 톤 이탈 | 톤 이탈은 자가 판정 불가 → 표본 3개를 사람에게 제시 후 진행 |
- 성공 반환 = `파일·건수` 1줄.

## 4. 실수 → 규칙 (append-only)
- [시드] 형식 지시 없이 생성해 파싱 실패 → "오직 JSON, 서두 없이"는 생략 불가 문구
- [시드] 전량 재생성으로 토큰 낭비 → 실패 필드/항목만 부분 재생성
- [시드] 생성만 하고 빌드에 안 넣어 제출 빌드에서 콘텐츠가 비어 있음 → §2-5 빌드 반영 확인이 done 조건
- [시드] "매 플레이 다르게"를 런타임 LLM 호출로 구현하려 함 → 사전 대량 생성 + 런타임 셔플로 대체
