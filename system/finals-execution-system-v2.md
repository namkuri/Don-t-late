# 본선 실행 시스템 v2 (Finals Execution System)

> **한 줄 정의** — 뼈대(좌표계)를 먼저 세우고, BOM(빈 소켓 지도)에 맞춰, 공정을 분리해(통짜/모듈) 병렬 생산한 뒤, 도착한 에셋을 미리 박힌 소켓에 **팍팍 꽂는** 시스템. 설계 목표는 셋: **의도 정렬 · 무낭비 · 병합 병목 제거.**

- **프로젝트**: 심야배송 (Midnight Delivery) · NAN 2026 본선 (48h · 1인/소수)
- **문서 성격**: 마스터 시스템 명세 (모든 하위 문서의 상위 골격)
- **버전**: v2 — 뼈대 초탄 · 소켓형 BOM · 공정분리(cast/module) · 태스크 흐름 모델(DES) 반영

---

## 0. 읽는 법 — 6가지 전제

### 0.1 두 시간대 — 레일 vs 콘텐츠

| 부류 | 무엇 | 언제 |
|---|---|---|
| **레일 (Rails)** | 주제 무관 인프라·규칙·템플릿 | 주제 공개 전 (25일 준비) — 파이프라인 배선, 공정분리 규칙, 오케스트레이션 규약, 에이전트 정의, 문서 템플릿 |
| **콘텐츠 (Content)** | 주제별 실제 내용 | 주제 공개 후 (48h) — 뼈대 본문, BOM, 태스크·진척, 실행 |

### 0.2 문서 우선순위 (충돌 시 상위가 이김)

```
INTENT.md > SCOPE.md > ARCHITECTURE.md > ADR/* > GDD.md > MDA.md
(왜·톤)     (범위)      (뼈대 규약)        (결정)    (무엇)    (재미)
```

### 0.3 시스템 불변식 (전역 규칙)

1. **단일 진실 원천** — INTENT/SCOPE 위배 산출물은 통과 불가.
2. **문서 없는 실행 없음** — 모든 작업은 `TASKS.md` 항목에서만 시작.
3. **뼈대 없는 양산 없음** — §1.5 뼈대 완료 전 에셋 양산 금지. *(신규)*
4. **검증 없는 통합 없음** — reviewer 통과 없이 씬·메인 반영 금지.
5. **스코프 락 이후 추가 없음** — 변경은 §0.6 프로토콜로만.
6. **되돌릴 수 없는 행동엔 사람 게이트** — 커밋·삭제·라이선스 반입·아트 확정·**통짜(cast) 발주**.
7. **2회 실패 → 자동 중단·에스컬레이션.**
8. **AI 활용은 그때그때 증거 로그에 기록.**

### 0.4 세 척추 개념 (v2의 핵심)

- **뼈대(Skeleton)** — 디렉토리·아키텍처 규약·데이터 스키마. **모든 후속 산출물이 꽂힐 좌표계.** 가장 먼저.
- **소켓형 BOM** — 에셋을 "만드는 정보"뿐 아니라 "**어디에 어떻게 꽂히는지**"까지 명세. 도착=교체.
- **공정분리(cast/module)** — 얽힌 것은 통짜로 찍어 병합을 *제거*, 독립적인 것은 모듈로 병렬 생산해 병합을 *사소화*.

### 0.5 단계 표기법

`입력 → 처리 → 산출물(파일) → 완료 정의(DoD) → 게이트 → 오너`

### 0.6 변경 프로토콜

INTENT/SCOPE/ARCHITECTURE 변경 시 → ADR 기록 → 영향받는 BOM·TASKS 항목 `stale` 표시 → 재검증 전까지 다운스트림 통합 중단.

## 1. 단계 1 — 기획 의도 파악 및 문서화

의도를 추론하지 말고 **강제 추출**한다. 어긋나면 아래 전부가 헛돈다.

| 칸 | 내용 |
|---|---|
| 입력 | 현장 주제 + 디렉터 |
| 처리 | 10분 의도 인터뷰(객관식) → MDA(재미 분해) → GDD(대상 정의) → ADR(결정·기각) |
| 산출물 | `INTENT.md` · `MDA.md` · `GDD.md` · `ADR/0001-*.md` |
| DoD | 에이전트에 그대로 던져 **되묻지 않고 실행 가능**할 것 |

```yaml
must_have: [ ... ]

ai_axis: "<AI 공정 레버리지 2개 이상 — 빌드타임에 AI를 어떻게 부렸나. 런타임 AI 아님>"
```

## 1.5 단계 1.5 — 뼈대 구축 (Skeleton) 〔신규·핵심〕

> **초탄은 "돌아가는 게임"이 아니라 "모든 게 꽂힐 빈 구조"다.** 뼈대가 좌표계를 깔아야 BOM이 소켓을, 공정분리가 인터페이스를 정의할 수 있다. 그래서 BOM보다 **먼저**.

| 칸 | 내용 |
|---|---|
| 입력 | GDD 코어 루프 |
| 처리 | ① 디렉토리 구조 ② 아키텍처 규약 ③ 데이터 스키마 확정 + ④ 빈 WebGL 관통 빌드 |
| 산출물 | 디렉토리 트리 · `ARCHITECTURE.md` · `data/schemas/*` · 배포된 빈 씬 링크 |
| DoD | 씬에 placeholder 소켓 배치 + 매니저/인터페이스 골격 컴파일 통과 + 데이터 스키마 확정 + **링크가 실제로 열림** |
| 게이트 | 사람: 규약이 SRP·컴포넌트 원칙을 만족하나 / 빌드 관통 확인 |
| 오너 | 사람 + 오케스트레이터 (규약 결정 = ADR) |

**세 구성요소**

1. **디렉토리 구조 = 소켓 보드.** 도착할 에셋·코드의 자리. BOM의 `dest` 경로와 일치해야 함.
   ```
   assets/{props,characters,env,audio}/   scripts/{systems,managers,interfaces}/   data/{schemas,content}/
   ```
2. **아키텍처 규약 = 모듈 인터페이스 계약.** 모듈 에이전트들이 서로 몰라도 같은 규약을 지켜 만들면 무충돌 조립된다.
   ```
   - 모든 상호작용체 → IInteractable 구현
   - 시스템 = 단일책임 매니저로 분리 (StaminaManager, TimerManager, DeliveryManager …)
   - 매니저 간 통신 = 이벤트만 (직접 참조 금지)
   - 에셋 = 프리팹 + 컴포넌트 조립 (상속 대신 조립)
   ```
3. **데이터 스키마 = 데이터 모듈 접합면.** 배송메모·후기·아이템·이벤트 스키마가 확정돼야 콘텐츠 에이전트가 LLM으로 찍어낸다 (국면 1~2에 **미리** 찍어 CSV/JSON으로 굽는다 — 런타임 생성 아님).

> **불변식 3** — 뼈대 미완성 상태에서 에셋 양산을 시작하면 인터페이스 불일치로 재작업 지옥. 국면 게이트로 강제 차단(§8).

## 2. 단계 2 — 소켓형 BOM

> BOM은 목록이 아니라 **빈 소켓의 지도**다. 씬에 placeholder를 먼저 박아두고, 에셋 도착 시 정해진 소켓에 **교체(swap)**만 한다. 배치가 고민이 아니라 조회(lookup)가 된다.

| 칸 | 내용 |
|---|---|
| 입력 | 코어 루프 + 뼈대(디렉토리·규약·스키마) |
| 처리 | 필요 에셋 전수 나열 → 각 항목에 산출·수용·공정·의존·최소화 정보 부여 |
| 산출물 | `ASSET_BOM.md` (또는 `.json`) |
| DoD | 코어 루프 전 장면 재생 시 누락 0 + 모든 항목에 placeholder 소켓 존재 |
| 게이트 | 사람: 장면 상상 재생하며 누락·소켓 점검 |
| 오너 | 사람 + 오케스트레이터 |

```yaml
# BOM 항목 스키마 (v2 — 소켓·공정·인터페이스 필드 추가)
- id: prop_porter
  fabrication: module          # cast | module (§2.5 공정분리)
  interface: IInteractable      # 준수할 아키텍처 규약 (§1.5)
  source: generate             # generate|reuse|fake|build
  tool: Meshy
  dest: assets/props/porter.glb # 뼈대가 깔아둔 경로
  placeholder_id: gray_porter   # 씬에 선배치된 소켓 (도착 시 교체)
  scene_slot: {parent: Complex_A, pos: [12,0,4], scale: 1.0, rot: 0}
  depends_on: []               # 위상 정렬 (§7)
  min_strategy: build          # 페이크>재사용>생성>제작
  eta_min: 12                  # 처리시간 추정 (스케줄용)
  license: "Meshy Pro"
  status: todo
```

**도착 = 교체.** 산출물이 검증 통과하면 `placeholder_id`의 그레이박스를 실물로 스왑. 좌표·부모·스케일이 이미 `scene_slot`에 있으니 배치는 자동.

## 2.5 단계 2.5 — 공정분리 (Cast / Module) 〔신규·핵심〕

> 병합이 어려운 건 **통짜로 찍어 병합을 제거**하고, 병합이 쉬운 건 **모듈로 병렬 생산해 병합을 사소화**한다. 병합 병목 구간을 설계로 우회.

**판별 질문:** *"따로 만들어 붙이는 비용 > 통으로 만드는 비용?"* → 크면 `cast`, 작으면 `module`.

| 축 | **Cast (통짜)** | **Module (모듈)** |
|---|---|---|
| 대상 | 요소가 강하게 얽힘 — 씬 블록(건물·계단·조명 상대위치), 긴밀 상호참조 로직 | 경계 깨끗 — 소품·데이터·오디오·UI 위젯 |
| 병합 | **없음** (통으로 생성, 접합면 자체가 없음) | **기계적** (규약 준수만 확인) |
| 병렬성 | 낮음 (1 컨텍스트/worktree 독점) | 높음 (다수 병렬) |
| 타이밍 | **늦게** — 방향 확정(스코프락+코어루프) 후 | **일찍** — 병렬로 |
| 수정비용 | 큼 (재주조) | 작음 (개별 교체) |
| 담당 | 캐스팅 에이전트 (소수) | 모듈 에이전트 (다수) |

**라우팅:** BOM의 `fabrication` 필드로 자동 분기. `cast`는 캐스팅 레인(늦게), `module`은 모듈 레인(일찍 병렬).

> **왜 cast는 늦게?** 통짜는 재주조가 비싸다(기가캐스팅의 수리 불가성). 방향 안 잡힌 채 찍었다 뒤집으면 최대 낭비 → cast 발주는 사람 게이트(불변식 6).

## 3. 단계 3 — 에이전트 분할 및 진척 메모리

에이전트는 스테이트리스 → **진척은 파일에 산다.**

| 칸 | 내용 |
|---|---|
| 입력 | BOM (fabrication 분류됨) |
| 처리 | 공정별 에이전트 편성 + 칸반 진척 관리 |
| 산출물 | `.claude/agents/*.md` (레일) · `TASKS.md` (콘텐츠) |
| DoD | `TASKS.md`만 보면 전체 진척 파악 |
| 게이트 | 오케스트레이터: done 항목 산출물·검증 확인 |
| 오너 | 오케스트레이터 (사람은 blocked만) |

**에이전트 편성 (공정분리 반영)**

- **캐스팅 에이전트** — cast 대상 통짜 생성. worktree 독점, 무거움, 소수.
- **모듈 에이전트** (asset/data/audio/ui) — module 대상 병렬 양산. 표준 인터페이스만 준수.
- **reviewer** — 수용 기준 + **인터페이스 준수** 검증. haiku로 병렬.

```yaml
# TASKS.md 칸반
- task: "포터 트럭 생성"
  bom_id: prop_porter
  fabrication: module
  state: doing            # todo|doing|review|done|blocked
  agent: asset-module-1
  depends_on: []
  attempts: 1             # retry_cap=2
  artifact: null
  eta_min: 12
```

## 4. 단계 4 — 파이프라인 배선 (2 레인)

| 칸 | 내용 |
|---|---|
| 입력 | BOM 항목 (status=todo) |
| 처리 | fabrication에 따라 두 레인으로 라우팅 |
| 산출물 | 소켓에 꽂힌 프리팹 + 갱신 매니페스트 |
| DoD | 검증 통과 후 씬 정위치 존재 |
| 게이트 | 각 화살표 끝 검증 게이트 |
| 오너 | 에이전트 (정밀 위치·조명은 사람) |

```
CAST 레인 (통짜):
  [뼈대 위 통짜 생성] → [검증] → [씬에 통합]         ← 병합 없음, 늦게

MODULE 레인 (병렬):
  [독립 생성] → [후처리] → [인터페이스 검증]
                              → [placeholder 소켓에 SWAP]  ← 도착=교체, 일찍
                              │
                        정밀 위치·조명 = 사람 수동 (AI 안 씀)
```

**의존성 처리** — `depends_on` 위상 정렬로 순서 결정. 독립 항목만 병렬, 순환은 loop_guard 금지.

## 5. 단계 5 — 산출물 최소화 필터

모든 에셋 요청 앞에 붙는 판정 필터. BOM의 `min_strategy`로 자동 분기.

**페이크 > 재사용 > 생성 > 제작**

| 전략 | 질문 | 처리 | 비용 |
|---|---|---|---|
| fake | 트릭으로 속이나?(안개·앵글·코드트윈) | 스킵/트릭 | 최저 |
| reuse | 기성 무료·스타터로 되나? | 임포트만 | 낮음 |
| generate | AI로 뽑나? | 생성+검수 | 중간 |
| build | 처음부터 제작 | 수제작 | 최고 — 코어루프에만 |

> 코어 루프 **밖** 요소에 `build` 태그 = 낭비 신호 → 사람 재검토.

## 6. 단계 6 — 프롬프트 & 오케스트레이션

**6.1 프롬프트 계약** — `목표·입력·기대·수용기준·실패시` 5칸. 구체 수치 필수. ("문 열어"✕ → "X축 0.9m 슬라이드, 0.4초"○)

**6.2 오케스트레이션** — 동기 허브-앤-스포크(사람=허브, 오케스트레이터 1, 서브에이전트 3~5 병렬). 국면별 병렬 강도 변동(§8). 되돌릴 수 없는 지점만 사람 승인. retry_cap 2, loop_guard.

## 7. 태스크 흐름 모델 (Entity–Station–Resource) 〔신규〕

> 이 시스템은 이산사건 관점으로 정확히 모델링된다. 이 렌즈가 **무낭비를 정량화**한다.

| DES 요소 | 우리 시스템 |
|---|---|
| **엔티티** | 작업 단위(에셋·기능). 속성: priority, depends_on, fabrication, eta_min |
| **스테이션** | 파이프라인 단계 (생성→후처리→임포트→검증→배치) |
| **자원** | 에이전트·사람 (seize/release) |
| **큐** | `TASKS.md` todo |
| **이벤트** | 상태 전이 (todo→doing→review→done) |

**최적화 규칙**

1. **병목 식별** — 십중팔구 reviewer/사람. 여기에 자원을 몰고(병렬 reviewer), 자동 가능 기준은 스크립트화.
2. **제약이론(TOC)** — 시스템 속도 = 병목 속도. 병목 아닌 곳 **과잉생산 금지**(WIP 재고 낭비). 생성 속도를 검증 속도에 맞춘다.
3. **WIP 상한** — `max_doing: 5`. 반쯤 만든 재공품이 곧 병합 리스크.
4. **최장처리 우선(LPT)** — `eta_min` 긴 것(cast·캐릭터) 먼저 투입해 병렬로 굴린다.
5. **의존성 = 라우팅 조건** — `depends_on` 해소(선행 완료 신호) 전엔 hold.

## 8. 48시간 국면 계획

각 국면의 전환점은 **미리 정해진 시각**이다. "이제 뭘 하지"를 고민하는 순간을 없앤다.

| 국면 | 시간 | 활동 | 무엇이 병합되나 |
|---|---|---|---|
| **0 기획** | 0~1h | 기획 파이프라인(pipelines/planning.md): 해부→발산→수렴→결정화 | 스코프 카드 (안 쏨) |
| **1 뼈대+관통** | 1~3.5h | 디렉토리·아키텍처·스키마 + 빈 WebGL 배포 확인 + 코어루프 그레이박스 | **뼈대 = 좌표계** |
| **2 양산** | 3.5~20h | cast 통짜(늦게) + module 병렬(일찍), 4h마다 통합 체크포인트 | 소켓에 팍팍 (연속 병합) |
| **3 분위기** | 20~30h | 조명·오디오·게임필 (레이어 덮기) | 충돌 적음 |
| **4 동결·마감** | 30~45h | 기능 동결, 버그·폴리싱, 문서 PDF화 | 안정화만 |
| **5 리허설** | 45~48h | 제출 점검 (손 뗌) | — |

**관통 규칙**
- **씬은 국면 1부터 항상 플레이 가능** — 이후 모든 국면은 착탄·교체·레이어링만. 빅뱅 병합 원천 차단.
- **긴 eta 먼저(LPT)** — 캐릭터·cast를 국면 2 시작에 발사해 백그라운드로, 빠른 소품은 나중에 → 통합 지점에 동시 도착.
- **cast는 늦게, module은 일찍** — cast는 방향 확정 후(재주조 방지), module은 병렬로 선행.
